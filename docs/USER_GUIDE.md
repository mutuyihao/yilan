# 用户文档

Last updated: 2026-05-25

这份文档面向第一次使用扩展的用户，也适合用来确认当前版本的真实功能边界。

## 这个扩展能做什么

- 对当前网页提取正文并生成结构化摘要。
- 对长文自动分段，再汇总成最终结果。
- 对 Bilibili 视频页提取视频信息，按“B 站官方 AI 总结 -> 可用字幕 -> 视频标题/简介等页面信息”的顺序构建摘要来源。
- 基于当前摘要继续生成 `行动项`、`术语表`、`问答卡片`。
- 在侧栏中查看来源信息、页面策略、可信与控制状态。
- 查看、搜索、收藏、删除历史记录，并按站点聚合筛选。
- 复制当前摘要或阅读页里的 Markdown。
- 导出 Markdown。
- 在 Bilibili 字幕可用时导出字幕 JSON 或纯文本。
- 生成带来源链接的长截图分享卡。
- 在新标签页打开专注阅读页，获得更适合阅读的版面。

## 安装

1. 打开 Chrome、Edge 或其他 Chromium 浏览器。
2. 进入扩展管理页。
3. 打开“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择当前项目目录。

## 首次配置

点击浏览器工具栏中的扩展图标，会打开设置页。

设置页分为三个标签：

- `连接`
- `偏好`
- `入口`

### 自动保存规则

设置页现在默认自动保存：

- 文本输入项会在停止输入后延迟保存。
- 文本输入项在离开输入框时会立即保存。
- 复选框和下拉框会立即保存。
- 关闭 popup 或切换页面前，未完成的改动会再尝试 flush 一次。

日常使用里，不需要再先点“保存”再继续操作。

### 连接

这里用来配置模型服务。

主要字段：

- `配置方案`：可选，用来保存多套连接配置并快速切换。`当前配置（未绑定）` 表示只编辑当前设置；绑定到某个配置方案后，后续自动保存会同步更新该配置方案。可通过右侧 `⋯` 管理（新建 / 另存为 / 重命名 / 删除）。
- `Provider`：选择服务商，设置页会自动带出推荐协议、Endpoint Mode、Base URL 和模型名。
- `更改地址`：当服务商提供多个 Base URL 时切换路线，例如按量 API、Token Plan、OpenAI 兼容或 Anthropic 兼容。
- `高级兼容设置`：用于手动调整 `Provider` 协议类型、`Endpoint Mode` 和 `Base URL`。自定义兼容接口会默认展开。
- `API Key`
- `Base URL`
- `模型名称`
- `刷新`：拉取 OpenAI 兼容接口返回的模型列表，用于输入提示（需要先填写 `API Key`）。Anthropic / Claude 兼容接口目前不自动拉取模型列表，仍可手动输入模型 ID。

建议流程：

1.（可选）如果你需要保存多套连接配置，先用“配置方案”新建或另存为一个方案。
2. 先选 Provider 服务商。
3. 如需代理、私有网关或多地址套餐，点击“更改地址”确认 Base URL。
4. 填写 API Key。
5. 按需覆盖模型名称。
6. 等待自动保存后点击“测试连接”。

当前内置预设包括：`自定义兼容接口`、`OpenAI 官方`、`Anthropic 官方`、`DeepSeek`、`Gemini / Google`、`xAI / Grok`、`Qwen / 百炼`、`GLM / 智谱`、`MiniMax`、`Doubao / 火山方舟`、`Hunyuan / 腾讯混元`。

补充说明：

- `Gemini / Google` 当前走 OpenAI 兼容的 `Chat Completions`。
- `xAI / Grok` 当前默认走 OpenAI 兼容的 `Responses API`，也可手动切到 `Chat Completions`。
- `MiMo / 小米` 同时提供按量 API 和 Token Plan 地域集群。Token Plan 可在“更改地址”中选择 CN、SGP、AMS 三个地域，每个地域都有 OpenAI 兼容和 Anthropic 兼容地址；不同地域的 Key 不通用。
- `Base URL` 既可以使用 provider catalog 的推荐地址，也可以在高级兼容设置中填写根地址或完整 endpoint；设置页会显示结构化请求预览。
- `Endpoint Mode = 自动判断` 会在 OpenAI 兼容自定义网关中尝试可用接口；连接测试可能缓存成功的 endpoint mode，并在网关错误足够明确时自动补齐或去除 `Base URL` 末尾的 `/v1`。
- 如果“测试连接”失败，底部状态会显示简要错误；必要时可以展开“错误详情”查看更具体的诊断信息。

### 偏好

这里用来设置输出与默认行为。

可配置项：

- `额外系统要求`
- `自动翻译输出`
- `默认输出语言`
- `明暗模式`：`自动跟随系统`、`固定浅色`、`固定深色`
- `色彩方案`：`松石绿`、`雾蓝`、`岩茶棕`、`檀紫`，对应内部 palette：`jade`、`slate`、`copper`、`plum`
- `侧栏默认精简模式`
- `无痕模式`
- `默认写入历史`
- `允许分享卡`

注意：

- 无痕模式开启后，结果不会写入本地历史。
- 无痕模式不会阻止页面内容、视频元信息或字幕被发送到你配置的模型服务。
- 关闭“默认写入历史”后，新结果只停留在当前侧栏。
- 关闭“允许分享卡”后，长截图分享功能会被禁用。
- 开启“侧栏默认精简模式”后，每次打开侧栏都会自动进入精简布局；来源、可信控制和底部状态区域会被压缩，摘要正文区域会获得更多可用空间。

### 入口

这里用来控制右键菜单、快捷键和默认触发行为。

可配置项：

- `入口自动生成摘要`
- `入口默认简短总结`
- `优先显示本页历史摘要`
- `打开历史记录`
- `检查入口`
- `快捷键设置`

默认快捷键是 `Alt + S`。

如果快捷键没反应，可以先点“检查入口”，再打开浏览器的快捷键设置页确认绑定。

## 日常使用

### 1. 打开侧栏

你可以通过两种方式触发：

1. 右键页面，选择“用一览总结此页”。
2. 按 `Alt + S`。

### 2. 生成摘要

侧栏打开后：

- 如果开启了“入口自动生成摘要”，会直接开始生成。
- 如果开启了“优先显示本页历史摘要”，并且当前页面已有已完成记录，会先直接加载最近一条历史摘要。
- 如果关闭了它，只会打开侧栏，等待你手动开始。

同一页面内的 SPA 路由切换有单独的默认行为：

- 如果侧栏已经打开，路由切换后会刷新为新页面的来源信息和正文上下文。
- 路由切换不会自动开始新的模型请求；如果新页面没有可复用历史，侧栏会显示“页面已就绪”，等待你点击“重新生成”。
- 如果旧页面摘要正在生成，路由切换默认不会打断旧任务；旧任务完成、失败或取消后，侧栏再追上最新页面上下文。
- 如果新路由已经有可复用历史摘要，并且开启了“优先显示本页历史摘要”，侧栏可以直接显示该历史；这不属于自动生成，因为不会请求模型。

侧栏会显示：

- 来源信息
- 页面类型和页面策略
- 当前摘要模式
- 可信与控制状态
- 主要摘要内容
- 运行状态和基础诊断

侧栏有两种布局：

- 标准模式会完整保留来源、可信控制、操作区、诊断入口和状态提示，适合检查当前页面上下文。
- 精简模式会保留这些信息但压缩展示，把更多高度留给摘要正文。若在设置页开启“侧栏默认精简模式”，每次通过右键菜单、快捷键或 popup 入口打开侧栏时都会自动进入精简模式。

### 3. Bilibili 视频页

打开 `bilibili.com/video/BV...` 页面后，触发方式仍然是右键菜单或 `Alt + S`。一览会使用专门的视频来源抽取流程：

- 先读取视频标题、UP 主、分 P、时长、简介等元信息。
- 优先尝试 B 站官方 AI 总结；如果可用，主摘要直接使用官方总结，不额外请求你配置的模型服务。
- 如果官方总结不可用，会尝试读取可用字幕，优先选择中文或中文相关字幕，再交给你配置的模型服务生成摘要。
- 如果官方总结和字幕都不可用，会回退到视频标题、简介和页面可见信息。
- 侧栏诊断中会记录 Bilibili 来源阶段，例如 `view`、`nav`、`player`、`official_ai_summary`、`subtitle`。

如果字幕内容可用，侧栏操作区会显示 `导出字幕`。优先导出原始字幕 JSON；没有 JSON 时导出纯文本。

使用限制：

- 目前稳定识别的是 `bilibili.com/video/BV...` 视频页。番剧、直播、专栏、动态、收藏夹、搜索页等其他 B 站页面可能只会按普通网页处理，或无法得到视频级来源。
- 官方 AI 总结、字幕列表、分 P 信息和播放器数据都依赖 B 站页面/API 返回。接口变更、权限限制、地区限制、视频未开放相关能力或当前浏览器未登录时，可能无法取得这些数据。
- 如果官方 AI 总结可用，主摘要可以直接使用官方总结；如果进入字幕或页面信息回退，一览会把字幕或回退文本发送给你配置的模型服务来生成摘要。
- 字幕优先选择中文或中文相关轨道；没有中文轨道时会选择第一个可用字幕轨道。字幕内容可能不完整，也可能与视频实际音频存在偏差。
- 导出字幕按钮只在当前会话或历史诊断中存在可导出的字幕内容时显示。持久化历史会裁剪长字幕诊断预览，不保证保留完整字幕正文。

免责说明：

- 一览不是 Bilibili 官方产品，与 Bilibili 没有从属、授权、背书或保证关系。
- 官方 AI 总结、字幕文本和模型生成结果都可能存在遗漏、错误或时间线偏差，不能替代观看原视频，也不能作为官方转写、事实核验、法律、医疗、金融或其他专业建议。
- 请仅在你有权处理的范围内使用摘要和字幕导出，遵守视频版权、Bilibili 平台规则和原作者权益。未经授权不要公开再分发完整字幕、视频转写或可能替代原内容的材料。

### 4. 切换摘要模式

当前主摘要模式有：

- `简短总结`
- `标准总结`
- `详细分析`
- `关键要点`

二次生成模式有：

- `行动项`
- `术语表`
- `问答卡片`

主摘要通常用于“重新生成当前摘要”，二次生成用于“基于已有摘要继续加工”。
如果入口先加载了本页历史摘要，此时点击“重新生成”仍然会基于当前页面重新跑，不会拿旧记录里的正文快照重算。

### 5. 使用历史与收藏

侧栏顶部可以打开“历史/收藏”面板。

支持：

- 搜索标题、站点、模式、模型
- 只看收藏
- 按站点聚合查看
- 收藏 / 取消收藏
- 删除历史记录

如果当前结果被标记为“不写入历史”，它不会进入历史，也不能加入收藏。
如果开启了“优先显示本页历史摘要”，入口触发时命中历史会直接显示旧摘要；你仍然可以点击“重新生成”来更新当前页面。

### 6. 打开独立阅读页

当侧栏里已经有摘要内容时，可以点击顶部“阅读”按钮。

独立阅读页会：

- 在新标签页打开
- 左侧显示摘要来源和操作区
- 右侧显示正文阅读区
- 根据摘要 Markdown 标题生成右侧文档导航
- 让正文滚动不被顶部工具栏遮挡

适合长时间阅读和复制整理内容。

如果阅读页提示会话失效，回到侧栏重新点击“阅读”即可。

### 7. 导出与分享

当前支持四种主要输出：

- 复制当前摘要 / 阅读页 Markdown
- Markdown 导出
- Bilibili 字幕导出（仅在字幕可用时显示）
- 长截图分享卡

分享卡会附带来源链接。如果“允许分享卡”关闭，则分享功能会被拦截。

## 数据保存到哪里

### 浏览器同步存储

设置项保存在 `chrome.storage.sync` 中，例如：

- API Key
- 模型配置（厂商预设 / Provider / Endpoint Mode / Base URL / 模型名称 / 额外系统要求等）
- 配置方案索引与当前激活配置方案
- 明暗模式与色彩方案
- 隐私、侧栏默认精简模式与入口设置

### 浏览器本地存储

运行时本地状态保存在 `chrome.storage.local` 中，例如：

- 右键菜单 / 快捷键入口状态
- 模型列表缓存（按 Provider + Base URL 维度缓存，用于输入提示）
- 自动 Endpoint Mode 缓存（OpenAI 兼容接口自动判断成功后按 Provider + Base URL 维度缓存）
- 阅读页临时会话

### IndexedDB

历史记录保存在当前浏览器 profile 的 IndexedDB 中，数据库名为 `aiSummaryDB`，主 store 为 `summaryRecords`。

Bilibili 视频抽取的运行诊断会随历史记录保存；完整字幕正文只在作为摘要来源或当前会话导出时使用，持久化诊断会裁剪长文本预览。

## 常见问题

### 右键菜单或快捷键没有反应

- 先到设置页的 `入口` 标签点击“检查入口”。
- 再打开“快捷键设置”确认 `Alt + S` 是否已绑定。
- 改过 `content.js`、`sidebar.js` 或 `sidebar.html` 后，记得刷新目标网页。

### 设置改了但好像没保存

- 文本输入是延迟保存，不是每敲一个字就立刻写入。
- 离开输入框时会立即保存。
- 底部状态栏会显示自动保存状态。

### 测试连接后 Base URL 或 Endpoint Mode 变了

这是当前实现的自动兼容逻辑。部分 OpenAI 兼容网关对 `/v1` 前缀或 endpoint 类型要求不同；如果连接测试能判断出更合适的写法，会把结果同步回设置页，并把成功的自动 Endpoint Mode 缓存在本地，后续减少重复试探。

### 刷新模型列表没有返回模型

模型列表刷新目前只针对 OpenAI 兼容接口调用 `/models`。Anthropic / Claude 兼容接口会提示不支持自动拉取，手动填写模型 ID 即可。

### 生成成功了，但历史里没有

通常是以下原因之一：

- 开启了无痕模式。
- 关闭了“默认写入历史”。
- 当前结果本来就被页面级策略标记为不入历史。

### 无法打开阅读页

一般是当前还没有摘要内容，或者之前打开阅读页的临时会话已经过期。回到侧栏重新生成或重新点击“阅读”即可。

### 分享卡按钮不可用

先检查：

- 当前是否已经有摘要内容。
- “允许分享卡”是否开启。

### 某些页面无法提取

浏览器内部页面、扩展商店页面或其他受限制页面通常无法注入内容脚本，这是浏览器权限模型的限制，不是侧栏本身的问题。

### Bilibili 视频没有官方总结或字幕

这通常取决于 B 站是否给该视频提供官方 AI 总结、字幕、分 P 数据以及当前浏览器登录态。一览会自动降级到标题、简介和页面可见信息，但这种情况下摘要会更像“视频信息概要”，不是完整内容总结。
