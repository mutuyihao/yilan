# 用户文档

Last updated: 2026-04-03

这份文档面向第一次使用扩展的用户，也适合用来确认当前版本的真实功能边界。

## 这个扩展能做什么

- 对当前网页提取正文并生成结构化摘要。
- 对长文自动分段，再汇总成最终结果。
- 基于当前摘要继续生成 `行动项`、`术语表`、`问答卡片`。
- 在侧栏中查看来源信息、页面策略、可信与控制状态。
- 查看、搜索、收藏、删除历史记录，并按站点聚合筛选。
- 导出 Markdown。
- 生成带来源链接的长截图分享卡。
- 在新标签页打开专注阅读页，获得更适合阅读的版面。

## 安装

1. 打开 Chrome、Edge 或其他 Chromium 浏览器。
2. 进入扩展管理页。
3. 打开“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择当前项目目录。

## 首次配置

点击浏览器工具栏中的扩展图标，会打开设置页。

设置页分为三个标签：

- `连接`
- `偏好`
- `入口`

### 自动保存规则

设置页现在默认自动保存：

- 文本输入项会在停止输入后延迟保存。
- 文本输入项在离开输入框时会立即保存。
- 复选框和下拉框会立即保存。
- 关闭 popup 或切换页面前，未完成的改动会再尝试 flush 一次。

日常使用里，不需要再先点“保存”再继续操作。

### 连接

这里用来配置模型服务。

主要字段：

- `厂商预设`：给出更稳的默认 Provider、Endpoint Mode、Base URL 和模型名。
- `Provider`：当前运行时支持 `OpenAI / OpenAI 兼容接口` 与 `Anthropic / Claude 兼容接口`。
- `Endpoint Mode`：显式指定最终接口类型。
- `API Key`
- `Base URL`
- `模型名称`

建议流程：

1. 先选厂商预设。
2. 再确认 Provider 和 Endpoint Mode。
3. 填写 API Key。
4. 按需覆盖 Base URL 和模型名称。
5. 等待自动保存后点击“测试连接”。

当前内置预设包括：`OpenAI`、`Anthropic`、`DeepSeek`、`Gemini`、`xAI`、`Qwen`、`GLM`、`MiniMax`、`Doubao`、`Hunyuan`。

补充说明：

- `Gemini / Google` 当前走 OpenAI 兼容的 `Chat Completions`。
- `xAI / Grok` 当前默认走 OpenAI 兼容的 `Responses API`，也可手动切到 `Chat Completions`。

### 偏好

这里用来设置输出与默认行为。

可配置项：

- `额外系统要求`
- `自动翻译输出`
- `默认输出语言`
- `配色模式`
- `无痕模式`
- `默认写入历史`
- `允许分享卡`

注意：

- 无痕模式开启后，结果不会写入本地历史。
- 无痕模式不会阻止页面内容被发送到你配置的模型服务。
- 关闭“默认写入历史”后，新结果只停留在当前侧栏。
- 关闭“允许分享卡”后，长截图分享功能会被禁用。

### 入口

这里用来控制右键菜单、快捷键和默认触发行为。

可配置项：

- `入口自动生成摘要`
- `入口默认简短总结`
- `优先显示本页历史摘要`
- `打开历史记录`
- `检查入口`
- `快捷键设置`

默认快捷键是 `Alt + S`。

如果快捷键没反应，可以先点“检查入口”，再打开浏览器的快捷键设置页确认绑定。

## 日常使用

### 1. 打开侧栏

你可以通过两种方式触发：

1. 右键页面，选择“用一览总结此页”。
2. 按 `Alt + S`。

### 2. 生成摘要

侧栏打开后：

- 如果开启了“入口自动生成摘要”，会直接开始生成。
- 如果开启了“优先显示本页历史摘要”，并且当前页面已有已完成记录，会先直接加载最近一条历史摘要。
- 如果关闭了它，只会打开侧栏，等待你手动开始。

侧栏会显示：

- 来源信息
- 页面类型和页面策略
- 当前摘要模式
- 可信与控制状态
- 主要摘要内容
- 运行状态和基础诊断

### 3. 切换摘要模式

当前主摘要模式有：

- `简短总结`
- `标准总结`
- `详细分析`
- `关键要点`

二次生成模式有：

- `行动项`
- `术语表`
- `问答卡片`

主摘要通常用于“重新生成当前摘要”，二次生成用于“基于已有摘要继续加工”。
如果入口先加载了本页历史摘要，此时点击“重新生成”仍然会基于当前页面重新跑，不会拿旧记录里的正文快照重算。

### 4. 使用历史与收藏

侧栏顶部可以打开“历史/收藏”面板。

支持：

- 搜索标题、站点、模式、模型
- 只看收藏
- 按站点聚合查看
- 收藏 / 取消收藏
- 删除历史记录

如果当前结果被标记为“不写入历史”，它不会进入历史，也不能加入收藏。
如果开启了“优先显示本页历史摘要”，入口触发时命中历史会直接显示旧摘要；你仍然可以点击“重新生成”来更新当前页面。

### 5. 打开独立阅读页

当侧栏里已经有摘要内容时，可以点击顶部“阅读”按钮。

独立阅读页会：

- 在新标签页打开
- 左侧显示摘要来源和操作区
- 右侧显示正文阅读区
- 让正文单独滚动，不被头部遮挡

适合长时间阅读和复制整理内容。

如果阅读页提示会话失效，回到侧栏重新点击“阅读”即可。

### 6. 导出与分享

当前支持两种主要输出：

- Markdown 导出
- 长截图分享卡

分享卡会附带来源链接。如果“允许分享卡”关闭，则分享功能会被拦截。

## 数据保存到哪里

### 浏览器同步存储

设置项保存在 `chrome.storage.sync` 中，例如：

- API Key
- 模型配置
- 主题偏好
- 隐私与入口设置

### 浏览器本地存储

运行时本地状态保存在 `chrome.storage.local` 中，例如：

- 右键菜单 / 快捷键状态
- 阅读页临时会话

### IndexedDB

历史记录保存在当前浏览器 profile 的 IndexedDB 中，主 store 为 `summaryRecords`。

## 常见问题

### 右键菜单或快捷键没有反应

- 先到设置页的 `入口` 标签点击“检查入口”。
- 再打开“快捷键设置”确认 `Alt + S` 是否已绑定。
- 改过 `content.js`、`sidebar.js` 或 `sidebar.html` 后，记得刷新目标网页。

### 设置改了但好像没保存

- 文本输入是延迟保存，不是每敲一个字就立刻写入。
- 离开输入框时会立即保存。
- 底部状态栏会显示自动保存状态。

### 生成成功了，但历史里没有

通常是以下原因之一：

- 开启了无痕模式。
- 关闭了“默认写入历史”。
- 当前结果本来就被页面级策略标记为不入历史。

### 无法打开阅读页

一般是当前还没有摘要内容，或者之前打开阅读页的临时会话已经过期。回到侧栏重新生成或重新点击“阅读”即可。

### 分享卡按钮不可用

先检查：

- 当前是否已经有摘要内容。
- “允许分享卡”是否开启。

### 某些页面无法提取

浏览器内部页面、扩展商店页面或其他受限制页面通常无法注入内容脚本，这是浏览器权限模型的限制，不是侧栏本身的问题。
